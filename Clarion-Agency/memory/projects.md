# projects.md
# Clarion — Active Project Tracker
# Version: 1.0
#
# PURPOSE
# Tracks ongoing workstreams the office is actively advancing.
# Agents update the relevant project entry in their WORK COMPLETED block each run.
# Chief of Staff reads this file and reports project health in the CEO brief.
#
# RULES
# - Do not delete entries. Change Status to Completed or Archived instead.
# - Blocked projects must include a specific blocker description.
# - Escalate? = Yes means the Chief of Staff must surface this in CEO brief.
# - Agents update Next Step and Last Update fields each cycle if work advanced.
#
# CAPACITY SAFEGUARD
# Maximum Active Projects = 7.
# A project is "active" if its Status is NOT Completed or Archived.
# Agents may NOT create a new project entry if 7 active projects already exist.
# If an agent believes a new project is necessary, it must do ONE of the following
# before proposing the new project:
#   1. Recommend closing (marking Completed or Archived) an existing project
#      that is no longer delivering value — name it specifically.
#   2. Recommend merging the new workstream into an existing project — name
#      the target project and explain the merge rationale.
#   3. Escalate to the CEO with: (a) the proposed new project, (b) the existing
#      project it would displace, and (c) a one-sentence justification.
# An agent that creates a new project entry without following this protocol
# is in violation of standing orders. Chief of Staff will flag it.
