# prelaunch_activation_mode.md
# Clarion Agent Office — Operating Priority State
# Version: 1.0 | 2026-03-12

---

## CURRENT STATE: PRE-LAUNCH ACTIVATION MODE

This is not demo mode. This is not analysis mode. This is commercial execution mode.

Until launch readiness is confirmed (launch_readiness score >= 8/10 sustained across
two consecutive monthly reviews), every meaningful run must produce real pre-launch
commercial work — not commentary on commercial work.

---

## PRIORITY ORDER (enforced every run)

1. **Pipeline growth** — new researched, ICP-qualified prospects added
2. **Outreach generation** — drafted messages ready for founder send approval
3. **Pilot creation** — firms moved toward or through a pilot analysis
4. **Proof and narrative improvement** — proof assets activated, messaging tightened
5. **Site and product conversion improvement** — specific, bounded changes proposed
6. **Market visibility preparation** — content drafted, angles queued

---

## ACTION-FIRST RULE

When an agent can either (a) analyze a situation or (b) produce a commercial artifact
from that analysis — it must do both, in that order. Analysis that produces no artifact
is acceptable only when data is genuinely missing. Analysis that produces no artifact
when the data exists to act is a failed run.

---

## MINIMUM EXPECTED OUTPUTS PER ACTIVATION-MODE RUN

Every full office run must produce at least the following. These are floor minimums,
not ceilings.

| Output | Owner | Minimum |
|--------|-------|---------|
| New researched leads OR qualified lead updates | Outbound Sales | 1 or more |
| Outreach drafts OR follow-up drafts | Outbound Sales | 1 or more |
| Content queue additions | Pre-Launch Content | 1 or more |
| Product experience / conversion finding | Product Experience | 1 or more |
| company_state.md updated | Chief of Staff | Required every run |

Content queue additions and product findings may be skipped ONLY when the relevant
agent did not run this cycle. When the agent runs, these outputs are mandatory.

---

## WHAT DOES NOT COUNT AS ACTIVATION OUTPUT

- A report that summarizes a situation without adding leads, drafts, or queue entries
- A gap analysis with no corresponding action taken this run
- A recommendation to do something next run that could be done this run
- A "no activity" entry when public-source data was available to act on

---

## EXIT CONDITION

Activation mode ends when the CEO explicitly sets a new operating state in this file.
Until then, all agents and the Chief of Staff apply these rules as standing orders.
