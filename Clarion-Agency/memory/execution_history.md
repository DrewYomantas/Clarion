# execution_history.md
# Clarion — Execution History
# Version: 1.0
#
# PURPOSE
# Tracks major initiatives that have already been analyzed, proposed, or launched.
# Agents must check this file before proposing any new project, initiative, or
# major strategy shift. If a similar item already exists here or in memory/projects.md,
# agents must update, refine, or advance the existing item — not create a duplicate.
#
# RULES
# - Append-only. Never delete or overwrite entries.
# - Agents read this file as part of their Operating Model check on every run.
# - New entries are added by the Chief of Staff when a new initiative clears
#   the deduplication check and is worth tracking.
# - Entry status: Active | Completed | Superseded | Paused
#
# DEDUPLICATION TEST
# Before proposing a new initiative, an agent must ask:
#   1. Does this appear in execution_history.md (same or substantially similar goal)?
#   2. Does this appear in memory/projects.md (same or substantially similar goal)?
# If YES to either: do not propose a new item. Instead, reference the existing
# entry and describe how you are advancing or refining it.
# If NO to both: the initiative is new and may be proposed.
