# first_run_success.md
# Clarion Agent Office — First-Run and Early-Run Success Criteria
# Version: 1.0 | 2026-03-12

---

## PURPOSE
Defines what a successful activation-mode run looks like before pilots, customers,
or substantial proof assets exist. Sets minimum expectations without requiring
what hasn't been built yet.

---

## ACCEPTABLE GAPS ON RUN 1

The following are acceptable on early runs and do not constitute failure:
- Zero completed pilots
- Zero proof assets in proof_assets.md
- Empty or thin leads_pipeline.csv at run start
- No prior outreach history
- No content previously posted

These are starting conditions, not excuses for inaction.

---

## WHAT IS NOT ACCEPTABLE ON ANY RUN

A run with no commercial or product output is not acceptable.

Specifically, a run fails if ALL of the following are true:
- No new leads added to leads_pipeline.csv
- No lead rows updated with progress or qualification
- No outreach or follow-up drafts produced
- No content queue entries added
- No product experience finding filed

If public-source prospect data exists (it always does), the Outbound Sales agent
can always add leads. This means the first bullet above can always be satisfied.
A full failure across all five dimensions is never justified.

---

## SUCCESSFUL FIRST / EARLY RUN PRODUCES

At minimum, a successful early run contains:

| Artifact | File | Agent |
|----------|------|-------|
| Researched leads appended | data/revenue/leads_pipeline.csv | Outbound Sales |
| Outreach drafts created | reports/sales/outbound_sales_YYYY-MM-DD.md | Outbound Sales |
| Content queue entries | data/growth/content_queue.md | Pre-Launch Content |
| Product experience finding | memory/product_experience_log.md | Product Experience |
| company_state updated | memory/company_state.md | Chief of Staff |

Proof assets, pilot reports, and named testimonials are stretch goals on early runs —
not requirements. The five artifacts above are the floor.

---

## CHIEF OF STAFF EVALUATION ON EARLY RUNS

On runs 1-3, the Chief of Staff should evaluate primarily on:
- Were commercial artifacts produced? (leads, drafts, content)
- Were any hard blockers identified that need founder intervention?
- Is the office running in activation mode or passive reporting mode?

Do not penalize early runs for thin data or empty proof files.
Do penalize early runs for producing reports with no downstream artifacts.
